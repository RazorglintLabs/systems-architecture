Scope matters: each law applies to the systems and paths where the corresponding proof surface exists. | Principle | Enforcement posture | |-----------|---------------------| | **Separation of authority** | The system that runs the operation should not be the same system that approves the operation. | | **Fail-closed** | Covered invalid-policy, invalid-state, or broken-chain paths stop instead of widening permission. | | **Deterministic control** | Covered control paths prefer deterministic behavior: same inputs, same outputs. | | **Local-first** | Core proof paths are built to run without mandatory cloud services. | | **Deny-by-default** | Covered capabilities start denied and require explicit allowance. | | **No shared core** | Sovereign systems avoid a shared hidden runtime; integration happens through contracts and artifacts. | | **Append-only evidence** | Records are hash-chained where implemented and signed where implemented; deferred signing is named, not hidden. | --- ## Stack | Layer | Technology | |-------|-----------| | **Languages** | Python 3.11+, Rust 2021, TypeScript, PowerShell | | **Frameworks** | FastAPI (API surfaces), Flask/Jinja (local UI), React, Electron | | **Cryptography** | SHA-256 hash chains, Ed25519 signatures where implemented, AES-256-GCM sealed payloads | | **Storage** | SQLite, append-only JSONL ledgers, file-based sealed bundles | | **AI / Retrieval** | FAISS, sentence-transformers, structured multi-phase reasoning | | **Infrastructure** | Docker, local-first defaults, no mandatory cloud services for core proof paths | --- ## For Buyers and Decision-Makers **What you get:** systems and proof artifacts designed to make automation behavior inspectable — not just asserted. **What that means:** - Covered automation paths can show what happened, when, and under what authority. - AI reasoning can be kept separate from execution authority. - Regulatory evidence can be compiled from sealed artifacts and mapped to article-level requirements. - Operational records can be made tamper-evident through hash chains, receipts, and verifier surfaces. - Public claims can be checked against evidence before outreach, investor material, or buyer summaries ship. **What it does not mean:** - This is not "AI compliance in a box." - This is not a blanket security certification. - This is not a claim that every system is production-ready. - This is not a SaaS-only model;
